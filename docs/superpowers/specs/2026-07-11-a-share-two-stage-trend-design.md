# A股两阶段趋势选股与回测设计

## 目标

将用户标注的日线经验规则量化为可解释的选股策略：先在强势启动或趋势初起时进入关注池，再在日K收盘确认转强时买入。目标是让量化买点尽量接近用户从下往上标注的箭头，而不是仅拟合已上涨股票的事后走势。

策略使用本地 DuckDB 中的日线数据运行，面向全A股，排除名称包含 `ST` 或 `*ST` 的证券。

## 交易时序与数据边界

- 所有关注、买入和卖出判断均仅使用当日收盘后可得的 OHLCV、涨跌停价和技术指标。
- 买入信号于日K收盘确认，下一交易日开盘成交。
- 卖出信号于日K收盘确认，下一交易日开盘成交。
- 若开盘停牌、无有效价格，或涨跌停导致无法合理成交，订单标记为未成交或延后，不使用理想成交价。
- 指标以复权日线计算；交易成交价使用同一数据口径。

## 选股逻辑

### 阶段一：关注信号

任一关注条件成立时，将股票放入观察池。观察池默认有效期为 60 个交易日；新的关注信号会刷新有效期和原因。

1. **强势K线**：收盘价不低于涨停价的 98.5%，或日涨幅不低于 6% 且成交量不低于 20 日均量的 1.8 倍。
2. **放量突破**：收盘价突破前 60 日最高收盘价，且成交量不低于 20 日均量的 1.5 倍。
3. **趋势初起**：收盘价上穿 MA60，且 MA20 当日不下降。

每个关注信号须保存触发日期、触发类型、涨幅、均线值与量比，供图形复核。

### 阶段二：买入信号

只有处于有效观察期且未持仓的股票参与买入评分。以下为默认规则，均可配置：

- 趋势基础：收盘价高于 MA10、MA20、MA60，且 MA10 不低于 MA20，当前 MA20 不低于 3 个交易日前的 MA20。
- 突破确认：收盘价突破此前 5 日整理区最高收盘价。
- 量能确认：成交量不低于 5 日均量的 1.2 倍。
- 不追高限制：收盘价不高于此前 20 日最高收盘价的 112%，以避免突破后过度加速的追高买入。

趋势基础为必需项。其余三项中至少两项成立则形成买入信号。输出必须包含各子条件、总分、关注来源和买入原因。单只股票已有持仓时不重复买入。

## 资金、仓位与退出

- 初始资金：200,000 元。
- 最大持仓数：10。
- 新开仓目标金额：当日账户权益的 10%，按 100 股向下取整；现金不足则不成交。
- 同日信号超过可用仓位时，按买入总分、量比、突破幅度降序选取。
- 不因新信号强制替换已有持仓；仓位释放后再处理新信号。

### 风控与止盈

- 初始止损：买入价下方 10%。
- 浮盈达到 1R（买入价上方 10%）后：止损线至少抬升至买入价。
- 浮盈达到 2R（买入价上方 20%）后：止损线取买入价与 `买入后最高收盘价 - 2 × ATR(14)` 两者中的较高者。
- 收盘价跌破当前止损线时产生卖出信号，下一交易日开盘卖出。
- 不设置固定止盈上限；最大持有天数是默认关闭的可选参数。

止损比例、1R/2R阈值、ATR周期、ATR倍数、观察期、均线周期、量比和突破窗口都进入策略配置，不在代码中写死。

### 默认参数表

| 参数 | 默认值 |
| --- | --- |
| 观察池有效期 | 60 个交易日 |
| 强势涨幅 / 20日量比 | 6% / 1.8 倍 |
| 放量突破窗口 / 20日量比 | 60 日 / 1.5 倍 |
| 买入均线 | MA10、MA20、MA60 |
| 整理突破窗口 / 5日量比 | 5 日 / 1.2 倍 |
| 最大追高幅度 | 此前20日高点上方 12% |
| 初始风险 R | 买入价的 10% |
| 保本与移动止盈触发 | 1R / 2R |
| 移动止盈 | 14日 ATR 的 2 倍 |

ATR 使用 14 日真实波幅的简单移动平均；其余窗口均只包含信号日之前和信号日的日线。

## 回测与校准

### 人工样本复核

使用用户提供的非ST样本图作为正样本参考。为每只股票导出带关注/买入标记的K线图和信号明细，以便人工判断量化买点是否靠近用户从下往上标记的买点。ST样本仅保留为形态观察，不参与打分、调参或全A绩效。

### 全A绩效验证

- 仅使用两年可用日线的时间序列切分：前 70% 用于选择少量预定义参数组合，后 30% 锁定参数验证。
- 不进行随机打乱和不使用未来收益作为选股输入。
- 交易成本（佣金、印花税、滑点）由配置提供，并同时报告成本前后结果。
- 输出交易明细、未成交订单、每日候选、资金曲线和参数对比。

关键指标包括：交易次数、胜率、平均盈利、平均亏损、盈亏比、Profit Factor、平均持仓天数、总收益率、最大回撤、单笔R倍数和仓位利用率。

## 实施边界

在既有 `mns.selector`、`mns.backtest`、`mns.pipelines` 与配置文件中做最小增量修改。第一版不引入机器学习、实时下单、行业轮动或外部数据源；重点是日线可解释规则、可配置回测和人工样本核对。

## 验收标准

1. 给定任一交易日，全A筛选结果不含ST证券，并可展示每只候选的关注与买入理由。
2. 信号不会使用信号日之后的数据，且成交时间均为下一交易日开盘。
3. 20万元、最多10持仓、整手买入、止损与ATR移动止盈在回测交易明细中可追溯。
4. 能输出用户样本的信号叠加图以及全A回测的胜率、盈亏比、收益和回撤。
5. 自动化测试覆盖关键阈值、时间边界、ST过滤、仓位上限和退出状态转换。
