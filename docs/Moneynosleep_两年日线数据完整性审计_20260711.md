# MoneyNoSleep 两年日线数据完整性审计

- 审计时间：2026-07-11
- 数据库：`data/duckdb/mns.duckdb`
- 审计区间：2024-07-11 至 2026-07-10
- 策略股票池：筛选缓存截至 2026-05-20 的非 ST 沪深 A 股，共 5,027 只（沪市 2,258 只；深市 2,769 只）
- 审计方式：DuckDB 全表统计，并用 AkShare/Sina 交易日历独立交叉核对

## 结论

数据在“交易日是否缺失”这一层基本完整，但还不能直接认定为可安全回测的数据集。

独立交易日历在审计区间内给出 484 个交易日，数据库包含全部 484 个交易日，没有缺少整天行情；数据库另多出 `2026-02-23` 一天非交易日占位数据。因此数据库共有 485 个日期。

必须先处理以下质量问题：

1. 删除非交易日 `2026-02-23` 的占位日线；该日全库 3,425 行的成交量和成交额均为 0，且未标记停牌。
2. 将 1,293 行 OHLC 全零或非正数据识别为停牌/无效行情，不能按价格 0 参与指标或成交。
3. 35,609 行空 OHLC 均来自 QMT，主要是 159 只新股上市前的占位行；应在首个有效交易日前剔除。
4. `adj_factor` 在 2,435,891 行中全部为空，`pre_close` 有 2,435,843 行为空。同步入口默认使用前复权，但数据库无法利用复权因子独立验证复权一致性。
5. 股票名称缺失 2,335,303 行，且 ST 状态不是历史逐日快照；当前“不要 ST”的过滤存在时点偏差。

## 个股缺行与停牌

按 484 个真实交易日建立 5,027 只股票的简单笛卡尔网格后，还有 161 个“个股-日期”没有日线，分布在 31 个日期。它们主要集中在个股连续停牌或区间末端，不能仅凭缺行直接认定为采集失败。

当前数据的主要问题是停牌表达不统一：

- 只有 24 行被明确标记为停牌；
- 1,293 行零价数据没有一行被标记为停牌；
- 35,609 行空价数据也没有一行被标记为停牌；
- 因而仅依赖 `is_suspended` 字段不可靠。

## 上市前占位数据

5,027 只股票都至少存在一条有效日线，但有 159 只股票的首个有效 OHLC 晚于审计起始日。这些主要是区间内上市的新股。

## 对现有回测的实际影响

优化回测 `20260711_190525_2c83ad11` 已受到数据质量影响：

- `600889.SH` 在 2024-11-04 至 2024-11-15 连续 10 天为全零 OHLC，但没有停牌标记；
- 回测在 2024-11-04 以 0 元错误卖出，形成约 -100.03% 的单笔收益和 -10.0031R；
- `2026-02-23` 的假交易日还会额外推进均线、关注窗口和持仓天数，可能改变节后信号。

因此此前优化版的 +45.63% 收益、31.98% 胜率、2.886 盈亏比和 1.357 Profit Factor 只能视为未清洗数据上的阶段性结果，不能作为最终可信指标。零价停牌错误使收益被明显低估，不能简单把该笔亏损加回。

## 后续数据清洗规则

进入正式回测前，建议固定采用以下最小规则：

1. 只保留独立交易日历中的日期。
2. `OHLC <= 0` 或 OHLC 全空且成交量为 0 时，按不可交易处理；持仓估值沿用上一有效收盘价。
3. 上市前空行直接剔除，不参与滚动均线和关注窗口计数。
4. 停牌日不推进技术指标，不触发止损、买入或卖出。
5. 复权口径统一为前复权，并记录可审计的复权方式或复权因子。
6. ST 过滤改为逐日状态；在历史状态未补齐前，报告中明确标注幸存者偏差和 ST 时点偏差。
