## 最值得借鉴的设计

1. 稳定的数据模型优先：先定义 `Candle / Instrument / StrategyConfig / SignalDecision / OrderPlan / BacktestTrade / BacktestReport / BacktestResult`，再让 UI、图表、报告、审计共用。
2. 策略和撮合分层：策略只输出信号，回测引擎负责成交、止损止盈、计费、资金曲线和统计。
3. 保留链路：`SignalDecision -> OrderPlan -> OpenPosition -> Trade`。这条链路非常适合后续从回测过渡到实盘程序化交易。
4. 单根 K 线路径判断：用 `open -> low/high -> high/low -> close` 近似判断同一根 K 内止损、止盈、动态保护谁先触发。
5. 动态保护规则：保本、锁 R、阶梯移动止损可以抽象成 `DynamicProtectionRule`，不要散落在策略里。
6. 审计导出：除漂亮报告外，还要导出资金曲线 CSV、操作日志 CSV、审计 JSON，方便复盘、对账和回归测试。
7. 图表协议：前端/Qt 图表尽量消费统一数据包：`candles + trades + indicators + equity_curve + drawdown_curve`。

## 迁移到 A 股时必须改造

- 交易日历、午休、停牌和节假日。
- 前复权/后复权/不复权口径统一。
- 涨跌停、一字板、排队成交和不可成交。
- A 股 T+1 卖出限制。
- 佣金、印花税、过户费、最低佣金分项计算。
- 成交量约束，不能假设无限流动性。
- 组合级资金、行业暴露、单票仓位上限和最大回撤停机。

## 当前项目后续落地方向
